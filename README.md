 Tecnologías utilizadas
- Kotlin
- Spring Boot
- PostgreSQL (Neon)
- Gradle

 Arquitectura
- Controllers: endpoints de la API
- Services: lógica de negocio
- Repositories: acceso a datos
- Models: entidades
- DTOs: transferencia de datos
- Security: configuración de seguridad

 Configuración
El proyecto utiliza variables de entorno en un archivo `.env` para la conexión a la base de datos.

 Ejecución
1. Configurar variables en `.env`
2. Ejecutar el proyecto con Gradle o desde IntelliJ
3. Acceder a los endpoints desde Postman o navegador
